# Builder Agent Runbook

## Phase 0 — Gate

Read the genesis, Product Contract and approved workflow. Until Turn 2 approval, limit work to documentation, scaffolding, architecture tests, schemas, manifests, test harnesses, reference slice and non-behavioural shell.

## Phase 1 — Orientation

Map current tree, scripts, migrations, RLS, direct Supabase usage, module boundaries, tests and legacy-coupled code.

## Phase 2 — Convergence plan

Classify current areas as retain, adapt, replace, remove, defer or unknown. Keep the repository runnable.

## Phase 3 — Foundation

Establish strict TypeScript, Result and identifiers, composition roots, module public APIs, import gates, envelopes, telemetry contracts and local Supabase test harness.

## Phase 4 — Reference slice

Complete authenticate → resolve project → create Work Thread → assign → acknowledge. Prove domain, application, adapter, RLS, browser and diagnostics before expansion.

## Phase 5 — Vertical chain

Build Work Thread → Deliverable → Submission → Dispatch → Receipt Evidence → Claim Requirement. Do not build all screens first.

## Phase 6 — Five canonical paths

Run Happy Path, Internal Revision, Delivery Rejection, Expiry/Overdue and Claim Gap.

## Phase 7 — Commissioning

Prove restart, outbox retry, process resume, projection rebuild, diagnostic pack and safe operator controls.

## Handoff

State objective, files, contracts, tests, results, assumptions, unresolved gates, evidence and next safe action.
